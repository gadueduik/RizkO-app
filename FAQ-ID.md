## 1. Pertanyaan Umum

### Q1.1: Apa itu RizkO?
RizkO adalah aplikasi asisten trading komprehensif berbasis multiplatform yang dibangun menggunakan Avalonia UI. Aplikasi ini dirancang khusus untuk membantu trader kripto dalam melakukan manajemen risiko, mencatat jurnal trading secara terstruktur, menganalisis performa perdagangan, membaca berita teraktual secara real-time, serta memantau grafik interaktif.

### Q1.2: Platform apa saja yang didukung oleh RizkO?
RizkO mendukung ekosistem Desktop maupun Mobile:
*   **Desktop**: Windows 10/11 (64-bit), macOS 11.0 (Big Sur atau versi lebih baru), dan berbagai distribusi Linux modern (seperti Ubuntu 20.04+, Debian 11+, dan Fedora 35+).
*   **Mobile**: Android 10 (Target SDK adalah Android 15/API Level 36, dengan dukungan minimum hingga Android 6.0/API Level 23) untuk arsitektur CPU `arm64-v8a` (perangkat fisik) dan `x86_64` (emulator).

### Q1.3: Apakah RizkO gratis untuk digunakan?
Ya, RizkO sepenuhnya gratis untuk digunakan. Tidak ada biaya berlangganan, batasan fitur (paywall), atau keharusan membuat akun. Anda dapat memberikan dukungan sukarela kepada pengembang melalui menu donasi jika merasa aplikasi ini bermanfaat.

## 2. Data, Privasi & Penyimpanan

### Q2.1: Di mana data jurnal trading saya disimpan? Apakah diunggah ke server?
RizkO dirancang dengan prinsip **Privacy-First & Offline-First** (Mengutamakan Privasi & Offline). Seluruh data transaksi jurnal, preferensi kalkulator, konfigurasi berita, dan log aplikasi Anda disimpan secara lokal di dalam penyimpanan internal perangkat Anda sendiri. Tidak ada data yang diunggah ke server eksternal maupun komputasi awan (cloud) pihak ketiga.

### Q2.2: Di mana saya bisa menemukan file database di perangkat saya?
RizkO menggunakan engine database tertanam **LiteDB** (NoSQL database lokal yang cepat). Lokasi penyimpanan file database tersebut bervariasi tergantung platform yang Anda gunakan:
*   **Windows**: `%LocalAppData%\RizkO\` (biasanya di `C:\Users\<NamaUser>\AppData\Local\RizkO\`), yang berisi file `RizkO_TradingJournal.db` dan `RizkO.db`.
*   **Linux**: `~/.local/share/RizkO/` atau direktori data standar XDG.
*   **macOS**: `~/Library/Application Support/RizkO/`.
*   **Android**: Di dalam folder data privat aplikasi yang aman: `/data/user/0/com.rizko.app/files/`.

### Q2.3: Apakah saya bisa melakukan cadangan (backup) atau memindahkan database ke perangkat lain?
Ya. Karena seluruh data disimpan dalam bentuk file lokal, Anda cukup menyalin file `.db` (terutama `RizkO_TradingJournal.db`) ke tempat penyimpanan yang aman atau ke perangkat baru. Pastikan aplikasi RizkO telah ditutup sepenuhnya sebelum menyalin file untuk menghindari kerusakan data (corrupted file).

### Q2.4: Apakah saya bisa mengimpor riwayat transaksi saya dari Excel?
Ya, RizkO menyediakan fitur Impor Excel yang memungkinkan Anda memindahkan catatan transaksi lama dari file spreadsheet langsung ke dalam database jurnal lokal. Fitur ini dapat diakses melalui menu pengaturan database di versi desktop maupun mobile.

## 3. Wi-Fi Synchronization (Sinkronisasi Tanpa Cloud)

### Q3.1: Bagaimana cara kerja fitur Wi-Fi Sync (Sinkronisasi Wi-Fi)?
Fitur **Universal Wi-Fi Sync** dirancang agar Anda dapat menyinkronkan data jurnal trading antara komputer (Desktop) dan ponsel (Android) secara langsung melalui jaringan lokal (Wi-Fi atau Hotspot) tanpa server pihak ketiga.
Satu perangkat akan bertindak sebagai **Host** (menyalakan server HTTP lokal di port 18346 dan memancarkan sinyal penemuan lewat UDP di port 18345) dan perangkat lainnya sebagai **Client**. Begitu Anda mengetuk tombol "Connect & Sync", kedua perangkat akan saling mencocokkan data terbaru (delta sync) secara langsung.

### Q3.2: Apakah saya memerlukan koneksi internet aktif untuk menyinkronkan perangkat saya?
Tidak. Proses sinkronisasi sepenuhnya berlangsung di dalam jaringan lokal (LAN) atau Wi-Fi Anda. Selama kedua perangkat terhubung ke router Wi-Fi atau hotspot ponsel yang sama, Anda dapat menyinkronkan data secara offline tanpa kuota internet.

### Q3.3: Bagaimana aplikasi menyelesaikan konflik data saat sinkronisasi?
RizkO menerapkan dua aturan utama untuk menyelesaikan perbedaan data antara dua perangkat:
1.  **Data Transaksi**: Menggunakan strategi **Last-Write-Wins (LWW)** berdasarkan stempel waktu UTC (`LastModifiedAt`). Perubahan terakhir yang Anda lakukan pada salah satu perangkat akan menimpa data lama di perangkat lainnya.
2.  **Gambar/Screenshot**: Jika kedua perangkat memiliki lampiran gambar yang berbeda untuk transaksi yang sama, perangkat penerima akan mencadangkan gambar lamanya terlebih dahulu dengan akhiran nama stempel waktu (contoh: `_backup_yyyyMMdd_HHmmss`) sebelum menerapkan gambar baru. Hal ini mencegah hilangnya bukti gambar trading secara tidak sengaja.

### Q3.4: Mengapa perangkat mobile saya tidak dapat mendeteksi host desktop?
Masalah ini biasanya disebabkan oleh konfigurasi firewall atau isolasi jaringan:
*   Pastikan kedua perangkat terhubung ke jaringan Wi-Fi/Hotspot yang sama.
*   Periksa apakah router Anda mengaktifkan fitur "AP Isolation" atau "Client Isolation" (yang melarang perangkat lokal saling berkomunikasi).
*   Pada Windows, pastikan Anda telah mengizinkan aplikasi RizkO melewati Windows Defender Firewall untuk jaringan Privat (Private) maupun Publik (Public).
*   Jika penemuan otomatis (UDP broadcast) gagal, Anda dapat memasukkan alamat IP Host dan Port secara manual pada aplikasi client di HP Anda.

## 4. Crypto News Reader (Pembaca Berita Kripto)

### Q4.1: Dari mana Pembaca Berita mendapatkan artikel beritanya?
News Reader di aplikasi RizkO mengambil feed berita RSS secara real-time dari situs berita kripto terkemuka dunia (seperti CoinDesk, CoinTelegraph, dll.). Berita yang berhasil diunduh akan disimpan ke database lokal (`RizkO_Articles.db`) selama maksimal 7 hari agar pemuatan serta pencarian kata kunci di kemudian hari berjalan sangat cepat dan instan.

### Q4.2: Mengapa terkadang beberapa berita gagal dimuat?
Situs berita sering kali menggunakan sistem anti-bot (seperti Cloudflare) atau pembatasan akses (*rate-limiting*). Untuk mengatasinya, RizkO dilengkapi dengan sistem jeda acak dan percobaan ulang (*retry*) otomatis. Jika situs berita sedang sibuk atau memblokir sementara, pembaca feed akan menunggu sejenak sebelum mencoba lagi. Koneksi internet yang tidak stabil juga bisa menyebabkan kegagalan pemuatan sementara.

### Q4.3: Bagaimana cara kerja sistem Penilaian Relevansi (Relevance Scoring) berita?
Saat Anda mengetik kata kunci pencarian, sistem pencari internal akan menilai kecocokan artikel berdasarkan frekuensi kata kunci, kesesuaian pada judul berita, serta kebaruan waktu terbit. Ini memastikan berita yang paling krusial bagi pasar tampil paling atas.

### Q4.4: Apakah saya bisa menyesuaikan topik berita dan kata kunci pencarian pilihan saya?
Ya. Anda dapat membuka menu **Settings -> News Topics (Intents)** untuk menambahkan kata kunci baru, menghapus topik yang tidak relevan, atau mengubah urutan prioritas topik berita. Anda juga bisa mengatur topik default saat aplikasi pertama kali dibuka serta jumlah berita per halaman.

## 5. Integrasi Grafik TradingView

### Q5.1: Bagaimana cara mengubah simbol koin atau timeframe pada grafik?
Widget TradingView di dalam aplikasi bersifat interaktif secara penuh. Secara default, widget memuat kontrak perpetual `BINANCE:BTCUSDT.P`. Anda cukup mengklik kolom pencarian simbol di pojok kiri atas grafik untuk mencari koin lain (seperti ETHUSDT, SOLUSDT, dsb.). Pilihan timeframe (1m, 5m, 1h, 1D) dan indikator teknikal (RSI, MACD, Moving Average) dapat Anda pasang langsung dari bar menu atas grafik.

### Q5.2: Mengapa grafik menampilkan halaman kosong atau memuat tanpa henti?
Widget TradingView menggunakan browser webview tersemat. Masalah pemuatan biasanya disebabkan oleh:
*   Ketiadaan koneksi internet aktif.
*   **Desktop**: PC Windows Anda membutuhkan runtime Microsoft WebView2. Windows 11 sudah menyediakannya secara bawaan, namun untuk Windows 10 versi lama, Anda mungkin harus mengunduhnya secara manual dari situs Microsoft.
*   **Android**: Aplikasi "Android System WebView" pada ponsel Anda harus diperbarui ke versi terbaru melalui Google Play Store.

### Q5.3: Apakah tema grafik menyesuaikan dengan tampilan aplikasi saya?
Ya, RizkO mendukung sinkronisasi tema secara dinamis. Apabila Anda mengubah preferensi tampilan aplikasi antara Tema Terang (Light Mode) dan Tema Gelap (Dark Mode) di menu Settings, grafik TradingView akan memuat ulang secara otomatis untuk menyelaraskan warna latar belakang, kisi-kisi, dan indikator skala grafik.

## 6. Kumpulan Kalkulator Trading & Manajemen Risiko

### Q6.1: Kalkulator apa saja yang tersedia di panel samping?
Panel kalkulator di sisi kanan jurnal trading menyediakan berbagai alat bantu hitung instan untuk mempersiapkan transaksi Anda sebelum masuk ke bursa:
*   **PnL Calculator**: Menghitung potensi profit/loss kotor maupun bersih (dikurangi estimasi komisi) serta persentase keuntungan modal (ROI).
*   **Target TP Calculator**: Menghitung target harga Take Profit yang presisi berdasarkan harga masuk (entry), leverage, dan target PnL yang diinginkan (dalam nominal USD atau persentase).
*   **Risk Management Calculator**: Menentukan ukuran posisi (*position sizing*) maksimal dan jumlah koin yang aman dibeli berdasarkan total modal, batas toleransi risiko (%), dan jarak harga Stop Loss.
*   **Leverage Calculator**: Menghitung rasio leverage yang dibutuhkan, margin minimum yang diperlukan, atau batas maksimal kuantitas koin.
*   **Commission Calculator**: Menghitung perkiraan biaya komisi masuk/keluar bursa.
*   **Comparison Calculator**: Membandingkan hasil simulasi posisi Long vs Short secara berdampingan untuk satu entry yang sama.

### Q6.2: Bagaimana cara kerja Kalkulator Manajemen Risiko / Ukuran Posisi (Position Sizing)?
Kalkulator ini membantu Anda membatasi kerugian modal. Anda cukup memasukkan Total Modal, persentase risiko yang siap ditanggung (misal 1% atau 2%), Harga Entry, dan Harga Stop Loss. Kalkulator akan mengeluarkan:
*   Nominal uang maksimal yang boleh hilang jika terkena Stop Loss (Risk Amount).
*   Ukuran posisi nominal maksimal yang diperbolehkan (setelah dikalikan leverage).
*   Jumlah unit koin maksimal yang boleh dibeli/dijual.

### Q6.3: Apa kegunaan tiga mode yang ada di Kalkulator Leverage?
*   **Leverage Result**: Menghitung kelipatan leverage yang dibutuhkan untuk membuka posisi dengan margin yang Anda miliki (hasil dibulatkan ke atas ke bilangan bulat terdekat).
*   **Required Margin**: Menghitung margin minimum dalam USD yang wajib disiapkan untuk membuka ukuran posisi tertentu dengan leverage yang Anda pilih.
*   **Maximum Quantity**: Menghitung batas kuantitas koin maksimal yang dapat ditransaksikan berdasarkan margin dan leverage Anda saat ini.

### Q6.4: Apakah saya bisa menyalin hasil kalkulator dengan cepat untuk ditempel di tempat lain?
Ya. Setiap output penting (seperti Target TP dan Leverage) dilengkapi dengan tombol salin instan (*Copy to Clipboard*). Tombol ini akan menyalin angka bersih (tanpa imbuhan huruf "x" atau "USD") sehingga Anda dapat langsung menempelkannya (paste) ke kolom terminal trading pada bursa (exchange) pilihan Anda.

## 7. Penyelesaian Masalah (Troubleshooting) & Dukungan

### Q7.1: Aplikasi crash atau menghabiskan sumber daya di latar belakang. Bagaimana siklus hidup proses dikelola?
Pada versi terbaru (v1.0.4), RizkO telah dilengkapi sistem manajemen siklus hidup aplikasi yang ketat guna mencegah port socket yang terkunci dan kebocoran memori (memory leak):
*   **Di Desktop**: Menutup jendela utama aplikasi akan langsung mematikan seluruh proses dan thread latar belakang .NET secara instan.
*   **Di Android**: Jika aplikasi ditutup atau dihancurkan (`OnDestroy`), sistem native akan otomatis melakukan pembersihan socket dan menghentikan proses aplikasi (`KillProcess`) dari RAM agar tidak ada pemborosan daya baterai dan memori di latar belakang.

### Q7.2: Mengapa tombol "Donasi" dinonaktifkan, dan bagaimana saya bisa mendukung pengembang?
Tombol donasi dinonaktifkan sementara di dalam kode sumber (`IsDonationEnabled = false` pada `MainMenu.axaml.cs`) demi alasan keamanan pengujian dan rilis awal. Jika fitur ini diaktifkan di masa mendatang, antarmuka akan menampilkan alamat transfer USDT (BEP-20, TRC-20, ERC-20) serta tautan donasi lokal (Saweria, Trakteer) dan PayPal. Saat ini, bentuk dukungan terbaik yang dapat Anda berikan adalah dengan mengirimkan umpan balik (*feedback*) serta laporan bug untuk pengembangan aplikasi yang lebih baik.
